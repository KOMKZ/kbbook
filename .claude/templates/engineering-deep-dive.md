# E01 工程深文模板

## 适用场景

- 一个机制需要从真实问题推到原理。
- 读者需要知道线上为什么会出事。
- 有代码、配置、命令或日志可以验证。

## 文章骨架

```text
0. 为什么要学这个
1. 真实场景:系统在哪里用到它
2. 核心心智模型
3. 从一个故障/慢请求/错误输出开始
4. 底层机制逐层拆开
5. 最小 demo 或命令验证
6. 常见陷阱
7. 速记总结
8. 下一篇
```

## 必填内容

### 0. 为什么要学这个

写 3 个真实工程场景,并明确不懂的后果。

### 1. 心智模型

用 Mermaid 或 SVG 画关系图。图必须解释:

- 谁产生数据。
- 谁保存状态。
- 谁触发变化。
- 哪个环节最容易出问题。

### 2. 血泪教训

必须给错误代码、配置、日志、SQL、命令输出或 benchmark 中的一种。

### 3. 验证

按 `modules/verification-adapters.md` 选择领域验证方式。

## 自检

- 没有一上来定义概念。
- 不是知识点清单。
- 有可观察证据。
- 有错误和正确版本对比。
- 结尾能指导排障或实作。
