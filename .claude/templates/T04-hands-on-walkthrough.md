# T04 — 手算演练式(Hands-On Walkthrough Mode)

> **核心承诺**:读者读完不仅"知道",还能**自己拿计算器复算每一步**;算完之后,**有"我刚刚亲手让机器学会了"的体感**。

---

## 一、什么时候用这个模板

### ✅ 必用场景

- 子问题包含"**怎么做到**"、"**具体怎么算**"、"**机制是什么**"
- 教学对象是**对该算法/概念不熟的小白**(包括你自己第一次学)
- 涉及**有限步骤的数学过程**(权重更新 / 梯度计算 / 注意力分数 / 采样 / 量化 / 反向传播单步等)
- 父文章里出现"信息密度过高 / 思维跳跃"的具体技术节点

### ❌ 不要用的场景

- **史观叙事**(走 T01 推导式)
- **横向多方案选型**(走 T03 调研)
- **代码实现为主**(走 T02 code-demo)
- 概念太大、无法用 1-3 个具体数字例子覆盖(改为拆成多个 T04 子文章)

## 二、铁律(违反就重写)

| # | 铁律 |
|---|------|
| 1 | **必须有一个具体场景**,**读者 5 秒内能说"哪里见过"**(苹果香蕉 / 楼梯灯 / 邮件分类 / 投票预测……),不能直接抛抽象公式或纯数学符号(如 XOR 真值表 / 二元函数 / 集合 A,B)。**抽象的"任务描述"不算场景**——必须是真实物体或日常情境 |
| 2 | **必须用真实数字**——不能 "假设 x = a"、不能 "考虑一般情况";数据可以是简化的整数,但必须给出来 |
| 3 | **每步运算必须写出**:写完公式 → 代入数字 → 算出结果。不能跳步、不能省略 |
| 4 | **必须有验证步骤**:训练 / 计算完后,把结果再走一遍所有样本,显式确认"全对" |
| 5 | **必须有几何 / 直觉视角**:数字是工具,真理解在几何 / 物理直觉 |
| 6 | **必须推广**:从 N=2 例子推到一般情况(高维 / 长序列 / 大数据) |
| 7 | **必须有反例 / 上限**:用同一手算思路体验"什么情况下这个算法不行"——这是真理解的标志 |
| 8 | **附录:给读者自己复算的指令**——明确说"按 1-2-3 步从头算,你应该得到 X" |

## 三、文章结构骨架

```
[T??-A??-S??] 标题:XXX 怎么做到 YYY:跟我手算一遍

> 父文章引用 + 阅读时间 + 「全程手算可验证」声明
> 本文要回答:具体怎么做?为什么这样做就对?

## 1. 给自己设一个小问题:(具体场景)
   - 引一个生活化的小问题
   - 给出 5-10 个具体样本(整数,易手算)
   - 画一个 ASCII 坐标图 / 表格让读者建立直觉
   - 用"人眼一眼看出"过渡到"机器看不出,需要算"

## 2. 把"X"翻译成数学
   - 编号 / 表示约定(类别编号、向量、阈值)
   - 几何性质(为什么这条直线 / 这个超平面有意义)
   - 完整公式(score = ...,prediction = step(...))
   - 一句话总结"整个过程其实就是 …"
   - 抛出关键悬念:"剩下问题是这几个数字怎么找到?"

## 3. 给出学习规则 / 更新规则(纯文字 + 公式)
   - 一句话直觉版("错了把 X 朝 Y 挪")
   - 严格公式
   - 简化系数说明(本例用 η=1)
   - 关键句:"下面我们全程手算"

## 4. 手算 — 初始化
   - 给出初始值

## 5-N. 手算 — 样本 1 / 2 / 3 / ...
   - 每个样本一节
   - 每节四块:**算分数** / **判定预测** / **算误差** / **更新权重**
   - 每块都用「```」代码块展示真实数字
   - 关键时刻(第一次错 / 权重出现负值)加直觉解释 quote

## N+1. 验证一下:用最终权重把所有样本重新跑
   - 完整表格:输入 | score 计算 | score | y_pred | y | 对错
   - 总结"全对"或"还差几个"

## N+2. 几何视角:那条"刀"长什么样
   - 决策边界方程
   - 画在 ASCII 坐标里
   - mermaid 训练流程图

## N+3. 推到 N 维 / 一般情况
   - 算法不变,只是维度高
   - 强调"理解 2 维就理解了 GPT 的 12288 维"

## N+4. 反例 / 上限(什么时候这个方法不行)
   - 给出反例(XOR / 数据不可分 / 梯度爆炸 / ……)
   - 用同样的手算思路体验"权重永远在样本之间打转"
   - 回呼父文章对这个反例的引用

## N+5. 一句话总结
   - 用 4-5 行高度浓缩
   - 涵盖:做法 / 学习 / 几何 / 上限

## 附:你可以自己验证
   - 明确步骤:1) 从初始值开始 2) 按顺序走样本 3) 每次 ... 4) 最终应该得到 X
   - 说明学习率系数等参数
   - 让读者真的能自己跑通
```

## 四、关键写法 do/don't

### ✅ DO

- 数字"刚好够用就好",不要 4 位小数;**优先整数 + 1-2 位小数**
- 数据**故意设计成 1-3 轮内收敛**——展示算法 work,不耗读者耐心
- 关键时刻(权重符号反转 / 收敛瞬间)用 `> 直觉:...` 框旁注
- mermaid 流程图放在算完之后做收束,**不放在开始**(读者还没见过实例)
- 表格用斜杠 `|` 严格对齐,score 计算列宽允许较长
- 反例(XOR 之类)放在"全部学会"之后,作为震撼对比

### ❌ DON'T

- 不能 "假设 w₁ = a,w₂ = b……" —— 永远用真实数字
- 不能 "由对称性,我们只需算第一项" —— 全部算出来
- 不能 "学习率 η 略去不影响逻辑" —— 显式说 η=1 并代入
- 不能跳过单调样本(对的样本) —— 哪怕只是 "score=85>0,err=0,不更新",也要写出来,**这是节奏感的一部分**
- 不能在数据不收敛时假装收敛 —— 要么换数据,要么诚实写明"再跑 K 轮"

## 五、模板自检清单(每篇定稿前)
