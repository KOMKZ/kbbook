# E05 排障诊断模板

## 适用场景

- 主题的核心价值是定位问题。
- 可以通过命令、日志、配置或指标观察。
- 读者需要形成排障路径。

## 文章骨架

```text
0. 事故现场
1. 先看现象,不要猜结论
2. 建立系统心智模型
3. 第一轮排查:排除最常见原因
4. 第二轮排查:定位关键状态
5. 根因解释
6. 修复动作
7. 预防和监控
8. 排障速查表
```

## 证据要求

至少包含两类:

- 命令输出。
- 日志。
- 配置。
- SQL/Explain。
- 指标。
- benchmark。

## 自检

- 每一步都说明为什么查这个。
- 命令输出被解释成人话。
- 修复动作能复现验证。
- 结尾有排障 checklist。
