# E09 Go 标准库深入用法模板

## 适用场景

- 标准库中某个 API/机制需要深入理解才能正确使用
- 不是"语法陷阱"（E07），不是"从零构建"（E08），而是"用对现有 API"
- 读者需要理解内部机制才能做出正确配置决策
- 有明确的"不用会出事"的工程后果

**典型主题**：http.Server 优雅关闭、database/sql 连接池、context 超时传播、json 流式解码、http.Client 连接池

**不适合**：
- 语法陷阱/错误用法型（用 E07）
- 从零构建组件型（用 E08）
- runtime 原理深挖型（用 E02）

## 文章骨架

```text
目录
背景需求 — 没有这个机制会怎样，真实后果
机制理解 — mermaid 时序/流程图 + 关键类型/方法表格
正确用法 — 渐进式：最小可用 → 加超时 → 加排空 → 生产级完整版
常见坑 — 每个坑 = 错误代码 → 后果 → 根因 → 正确写法
总结 — 知识点表格 + 配置速查表
完整可跑代码 + 运行输出
```

## 门禁自检

| # | 检查项 | 通过标准 |
|---|--------|---------|
| S1 | **目录** | 文章顶部有锚点目录 |
| S2 | **背景需求有后果** | 明确写出"不用这个机制会导致什么线上事故" |
| S3 | **有 mermaid 图** | 至少一张时序图或流程图解释内部机制 |
| S4 | **渐进式用法** | 至少三步：最简 → 加防护 → 生产级完整版，每步解释"为什么加这一层" |
| S5 | **坑有完整诊断链** | 每个坑 = 错误代码 + 实际后果 + 根因 + 修复代码 |
| S6 | **总结有速查表** | 末尾有可直接查阅的配置参数表或 Do/Don't 表 |
| S7 | **代码可独立运行** | 文末完整代码 `go run` 无误，输出和文中一致 |
| S8 | **聚焦一个 API** | 只讲一个标准库机制，不串联多个不相关的 API |

## 写作约束

### 背景需求

```markdown
## 背景需求

[一句话说清这个 API 做什么]。

**没有它的时候**：[怎么做 → 什么后果 → 具体症状]。

**读完这篇你能**：[可操作的能力]。
```

### 机制理解

```markdown
## 机制理解

[mermaid 时序图/流程图]

| 类型/方法 | 作用 | 关键行为 |
|----------|------|---------|
| [Type1] | [职责] | [关键行为描述] |
| [Method1] | [职责] | [关键行为描述] |
```

图必须解释"谁在什么时候做了什么"。

### 正确用法（渐进式）

```markdown
## 正确用法

### 第一步：[最小可用版本]

[代码块]

这一步解决了 [X] 问题，但还有 [Y] 隐患。

### 第二步：[加上防护]

[代码块]

加了 [某机制]，解决了 [Y]。但 [Z] 场景还没覆盖。

### 第三步：[生产级完整版]

[代码块]

现在 [X]、[Y]、[Z] 全部覆盖，这是生产环境应该用的版本。
```

至少三步，每步必须说明"加什么"和"为什么加"。

### 常见坑

```markdown
## 常见坑

### 坑1：[坑名]

**错误写法**：

[代码]

**后果**：[具体症状]

**根因**：[为什么]

**修复**：

[代码]
```

每个坑是一个完整诊断闭环。最少 2 个坑，最多 4 个。

### 总结

```markdown
## 总结

| # | 知识点 | 说明 |
|---|--------|------|
| 1 | [关键概念] | [一句话] |

### 配置速查

| 参数 | 推荐值 | 作用 | 设错的后果 |
|------|--------|------|-----------|
| [参数名] | [值] | [作用] | [后果] |
```

## 与 E07、E08 的区别

| 维度 | E07 陷阱型 | E08 构建型 | E09 标准库深入 |
|------|-----------|-----------|--------------|
| 核心问题 | "看起来对，为什么错了" | "需求怎么用 Go 特性实现" | "这个 API 怎么用对" |
| 驱动方式 | 错误→排查→修复 | 需求→特性→迭代 | 机制→渐进用法→避坑 |
| 代码组织 | 错误版 vs 正确版对照 | 多个版本逐版增强 | 同一用法逐层加防护 |
| 篇幅 | 200-400 行 | 400-600 行 | 400-600 行 |
