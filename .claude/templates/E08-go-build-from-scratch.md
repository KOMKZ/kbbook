# E08 Go 从零构建实战模板

## 适用场景

- "从零实现一个 X"型文章
- 需求明确，逐步迭代，每个版本加入一个新的 Go 语言特性
- 并发组件设计（限流器、缓存、连接池、singleflight 等）
- 需要读者建立"用 Go 特性解业务问题"心智模型的场景

**不适合**：
- 语法陷阱/常见错误型（用 E07）
- runtime 原理深挖型（用 E02）
- 多方案对比选型（用 T03）

## 文章骨架

```text
目录
背景需求 — 要做什么，关键约束是什么
构思：Go 特性→需求映射表
算法/概念背景 — mermaid 图 + 参数表格
版本推导:
  v0    基础算法（不涉及并发）
  v0.1  需求逼出第一个 Go 特性
  v0.2  新需求逼出更多 Go 特性
  v1.0  优化：Go 的编码习惯
版本总结 — mermaid 演进图 + Go 特性表格
心智模型 — 思考框架
完整可跑代码 + 运行输出
```

## 门禁自检

| # | 检查项 | 通过标准 |
|---|--------|---------|
| B1 | **目录** | 文章顶部有锚点目录，覆盖所有主要章节 |
| B2 | **背景需求明确** | 3-5 句交代清楚要做什么 + 关键约束（并发、超时、不依赖外部等） |
| B3 | **Go 特性映射表** | 开头有一张"需求→要解决什么→Go 里用什么"的三列表 |
| B4 | **算法/概念背景** | 有 mermaid 图或原理图 + 核心参数表格 |
| B5 | **每个版本标注 Go 特性** | 每版明确指出"用了 Go 的什么特性、为什么选它" |
| B6 | **版本演进有 mermaid 图** | 版本总结处用 flowchart 串联各版本 |
| B7 | **心智模型收尾** | 末尾给出可迁移的思考框架或设计步骤，不止于总结知识点 |
| B8 | **代码可独立运行** | 文末完整代码复制到 `main.go` 后 `go run` 无误 |
| B9 | **故事驱动** | 每个版本的升级有明确的"新需求/新问题出现"作为动机 |

## 写作约束

### 背景需求

```markdown
## 背景需求

你要为 [系统/场景] 实现一个 [组件名]。关键约束：

- **约束1**：[说明]
- **约束2**：[说明]
- **约束3**：[说明]

下面从这个需求出发，看 Go 的 [相关特性] 如何一层层叠出一个生产级 [组件名]。
```

### 构思：需求→Go 特性映射表

```markdown
## 构思：用 Go 的什么来解这个问题

| 需求 | 要解决什么 | Go 里用什么 |
|------|-----------|------------|
| [需求1] | [问题描述] | [Go 特性] |
| [需求2] | [问题描述] | [Go 特性] |

这几个 Go 特性各解决一层问题，叠起来就是一个 [组件名]。
```

### 算法/概念背景

```markdown
## [概念名] 是什么

一句话定义 + mermaid 流程图 + 核心参数表格。
```

### 版本推导

每版格式：

```markdown
## v{version}：[新需求] → [Go 特性]

需求描述 + 为什么上一版不够。

[代码块]

**这里用到了一个关键的 Go 语言特性**：[一句话 + 说明]
```

每版必须标注：
- 新需求是什么
- 新增了哪个 Go 特性
- 为什么选这个特性而不是别的

### 版本总结

```markdown
## 版本总结

[mermaid flowchart 演进图]

| 版本 | 新增 Go 特性 | 解决了什么 |
|:---:|------------|-----------|
| v0 | [特性] | [解决的问题] |
| v0.1 | [特性] | [解决的问题] |
```

### 心智模型

```markdown
## 心智模型：[领域]的思考框架

这篇文章不是教你 [具体知识点]——是给你一个框架：

1. [步骤1]
2. [步骤2]
3. [步骤3]

最后一句总结。
```

## 与 E07、E01 的区别

| 维度 | E07 陷阱型 | E08 构建型 | E01 工程深文 |
|------|-----------|-----------|------------|
| 结构 | 错误→根因→正确 | 0→0.1→0.2→1.0 | 事故→心智模型→机制→陷阱 |
| 驱动方式 | "出了什么 bug" | "需求逼出特性" | "线上为什么会出事" |
| Go 特性 | 对比错误用法 | 每版新增一个特性 | 底层机制 |
| 篇幅 | 200-400 行 | 400-600 行 | 800-1500 行 |
