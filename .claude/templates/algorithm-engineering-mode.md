# E04 算法工程化模板

## 适用场景

- 算法不是为了刷题,而是为了解决工程问题。
- 需要比较暴力版和优化版。
- 需要边界用例和复杂度解释。

## 文章骨架

```text
0. 真实业务问题
1. 输入输出定义
2. 暴力解:为什么自然但慢
3. 观察重复计算或结构
4. 优化思路被逼出来
5. 手算小样本
6. 代码实现
7. 复杂度与 benchmark
8. 边界用例
```

## 必填

- 小样本手算。
- 暴力解。
- 优化解。
- 复杂度推导。
- 至少 5 个边界用例。

## 禁止项

- 直接贴最优解。
- 只写 Big-O 不解释为什么。
- 没有业务语境。
