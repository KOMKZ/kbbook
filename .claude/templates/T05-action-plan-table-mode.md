# T05 操作实施表格模板

## 适用场景

- 学习计划 / 学习路线 / 训练营安排
- 操作教学 / 实施步骤 / SOP
- 每日任务 / 阶段计划 / 项目执行表

## 写作硬规则

- 直接写任务、动作、产出、验收;少解释原因。
- 读者不懂会问,正文不要替读者预设长篇解释。
- 表格按放大镜结构递进:总纲表格 -> 阶段细节表格 -> 具体实施表格。
- 每个任务必须有时间、动作、工具或材料、产出、验收标准。
- 禁止写空泛词:理解、掌握、熟悉、提升。必须改成可检查产物。

## 固定结构

```text
0. 目标和验收
1. 总纲表格
2. 阶段细节表格
3. 具体实施表格
4. 每日固定节奏
5. 产物清单
6. 验收表
7. 下一步
```

## 表格层级

| 层级 | 表格名称 | 作用 |
|---|---|---|
| 第 1 层 | 总纲表格 | 让读者 1 分钟看完整体路线 |
| 第 2 层 | 阶段细节表格 | 把每个阶段拆成任务包 |
| 第 3 层 | 具体实施表格 | 写到每天/每步怎么做 |

## 自检

| 检查项 | 标准 |
|---|---|
| 是否直接 | 每节开头 3 行内出现任务表 |
| 是否可执行 | 每行都有动作和产出 |
| 是否可验收 | 每行都有检查标准 |
| 是否少废话 | 解释段不超过全文 20% |
